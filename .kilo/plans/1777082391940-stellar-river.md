# Plan: Port TMP Redesign to Main Frontend

## Goal
Replace the current tab-based UI with the TMP redesign's sidebar+topbar layout, dark cinematic theme, and all views (Gallery, Table, Boards, Decks, Deck Composer, Upload). Preserve all existing Convex backend functionality and auth. Add backend support where the TMP's data model exceeds what currently exists.

## Architecture
- **Port strategy**: Convert TMP's Babel-in-browser JSX into proper TypeScript/React components in `src/`. Keep `TMP/` as a standalone design reference.
- **Styling**: Port TMP's CSS custom properties into `src/index.css` alongside existing Tailwind. Use a hybrid approach — Tailwind utilities for layout, TMP's CSS vars for the dark cinematic theme.
- **State**: Replace TMP's mock data / `localStorage` with Convex queries/mutations. Keep the `tweaks` panel and its `localStorage` persistence.
- **Routing**: Keep the existing hash-based routing (`#gallery`, `#table`, `#boards`, `#deck`, `#deck:<id>`, `#upload`).

## Phase 5: Backend Additions

### 5.1 Deck Schema Extensions (`convex/schema.ts`)
Add optional fields to `decks` table:
```typescript
templateId: v.optional(v.string()),
templateName: v.optional(v.string()),
scrollFx: v.optional(v.string()),
overlay: v.optional(v.number()),
blocks: v.optional(v.array(v.object({ id: v.string(), label: v.string(), on: v.boolean(), locked: v.boolean(), kind: v.string(), variant: v.string() }))),
palette: v.optional(v.array(v.string())),
fontFamily: v.optional(v.string()),
logline: v.optional(v.string()),
characterName: v.optional(v.string()),
characterBody: v.optional(v.string()),
```

### 5.2 New Convex Queries/Mutations
| Endpoint | Purpose |
|---|---|
| `api.decks.listWithMeta` | List user's decks with computed metadata (block count, updated time) |
| `api.decks.updateMeta` | Update deck title, subtitle, tag, template, palette, etc. |
| `api.decks.updateBlocks` | Update enabled/locked block state |
| `api.decks.updateSlides` | Rich slide content update (if needed beyond current `slides` array) |

### 5.3 Image Filters
The TMP sidebar filters by `group`, `genre`, `style`, `originalsOnly`, `hasSref`. Ensure these map to existing schema fields:
- `group` → `images.group`
- `genre` → needs a new field or reuse `category` with genre mapping
- `style` → needs a new field or reuse `category` with style mapping
- `originalsOnly` → `parentImageId === undefined`
- `hasSref` → `sref !== undefined`

**Decision**: Add `genre` and `style` as optional string fields to `images` schema, or compute them from tags/category. Recommend adding fields for clean filtering.

## Risk Areas

1. **Deck Composer size**: `TMP/deck.jsx` is 1100+ lines of complex JSX with GSAP + container queries. This will be the longest port. Consider splitting into sub-components early.
2. **TypeScript conversion**: TMP uses no types, no imports, window-scoped state. Every component needs interfaces for props and Convex data shapes.
3. **Tailwind vs. inline styles**: TMP uses heavy inline styles. Decide per-component whether to convert to Tailwind utilities or keep inline for the cinematic precision.
4. **GSAP dependency**: Already in `package.json`. Ensure ScrollTrigger is registered and cleaned up properly in React lifecycle.
5. **Performance**: Masonry gallery with many images + deck composer with ResizeObserver on every slide. Test with large datasets.
